# Intro to CLI

## CLIとは？

キーボードから文字を入力して操作するインターフェース.
GUIとは対の関係.日本ではよく*CUI*と呼ばれるが、専ら英語ではCLI.

## 何が良いのか

1. 速い、軽い

最初は直感的ではないものの、マウス移動やクリックがいらない.
GUIにリソースを割かないので実行が速い.

2. 自動化に強い

同じ作業を何百回もしたり、夜中に勝手にプログラムを動かしたり、ミスを減らすために様々なツールを導入したりできる.

3. 組み合わせられる

一つ一つのコマンドができることは少ないが、`|(パイプ)`を使うことで組み合わせることができ、
これが非常に強力. Unix哲学の思想.

4. 再現性が高い

コマンドは履歴に残る. 同じコマンドなら基本的に他人のPCでも同じ結果が出る.再現性の観点ではDockerも併用するとより強力.

5. リモート環境に強い

GUIでリモートサーバにつなぐのは非効率的だし、大抵CLI前提. SSH一本で戦える.

6. PC操作の本質が見える

ファイルやプロセス、ネットワーク、権限、OS操作の招待を隠さない. GUIはただの皮.

7. 環境を選ばない

Linux, macOS, WSL2, サーバ、コンテナ、どこでも基本同じコマンドが使える. 覚えたことが腐らない.

8. エンジニアの共通語である

基本的なCLIコマンドを覚えておくと話が速い.
